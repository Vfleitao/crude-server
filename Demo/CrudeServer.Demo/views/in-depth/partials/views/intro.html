Views

Views in Crude Server are just another type of response that emits bytes to the response directly.
The build in system uses HandleBars.net but you can implemt your own quite easily.

View Providers implement the following interface: 

public interface ITemplatedViewProvider
{
    public Task GetTemplate(string templatePath, object data);
}
As long as you implement this interface, you can build views in whatever way you see fit.

Currently Crude Server uses the HandleBars syntax in with a few built in extensions to build layouts and partial views
